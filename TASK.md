# Build your first API endpoint

### Goal
Build the smallest possible backend — a server with two JSON endpoints — call it from curl and your browser, and publish it to a public GitHub repository.

### Purpose
In the lecture you watched the request → response loop; now you stand on the server side of it. Once you've made a server answer a request, the loop stops being abstract — and publishing to GitHub is how every piece of work in this program is submitted, so we set that habit on day one. Beginners usually overthink this: the whole thing is ~20 lines
